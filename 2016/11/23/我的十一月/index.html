我的十一月

发表于 2016-11-23   |   分类于 心情随笔   |     |   阅读次数

今天来自己的主页里看了一下，上次更新博客还是10月28号，时间太快，一不留神一个月就没了🤕。趁着今天在单位早早的把手头工作给结了，就回顾一下这近一个月以来的搬砖生活，毕竟这月28号又要奔赴大山东的”GW技校”回个炉，不知到时候会是怎样的体验~~~

首先，完成了线路巡视系统移动设备管理模块的功能，实现了移动设备信息的管理包括增删改查、移动设备的定位展示，使用了Openlayer3做前端的地图展示，使用Ajax定时刷新坐标以从数据库中获取最新的位置信息、移动设备的巡视轨迹的展示回放。通过这个模块的开发，让我这个GIS专业的学沫对WebGIS/切片地图等的原理有了更深的认识、对HTTP应用协议也有了一定的了解离掌握还有很长的路要走。现在使用的数据还是我自己在数据库中模拟的，不知道当我”上学”回来后，真的移动设备接入数据库后，会表现如何，🙏🙏🙏。

接下来，领导又安排做了一个表箱巡检系统移动端Android开发的任务。上学的时候，做过两个Android的小项目，一个是“车辆轨迹管理”，一个是“基于HDFS的云存储”算是入了这个Android的大门，不过那时候你大爷还是你大爷，Google大爷还依然支持Android dev tools for Eclipse，现在只能拥抱Android Studio了。为了配置开发环境还把我的Netfits账号充了值，方便翻墙，心累！不过官方提供的Simulator是真**慢，珍惜生命，于是我下载了很火的Genymotion模拟器，号称快如真机，亲测确实不错。这里有几个关于Genymotion的坑说明一下：

  • 不翻墙的话Genymotion下载速度忒慢，单位50M光纤，下载它速度只有几kb
  • 在AS里装了Genymotion插件，并且启动了模拟器，但是^ + r运行，会提示找不到设备，这是因为没有设置ADB tools的路径，这里应该使用AS sdk文件下的ADB tools，而不是默认Genymotion自带的。

目前已经把界面UI完成了，在此过程中，收货颇丰：

  1. 对Android Studio的使用更加的熟练，不过还有很多快捷键仍然记不住；
  2. 知道按钮各种效果的制作方法；
  3. 弄明白了Android5.0 Material Design向低版本兼容的方法，即AppCompat.v7的使用；
  4. 对比Html的盒模型，对Android的layout布局方法更加清晰，基本可以熟练的做出自己想要的布局样式😌。

当然不清晰的地方还有很多：

  1. 对Soap协议没有很好的理解，现在可以使用kSoap2包调用WebService;
  2. 对不同分辨率设备的适配方法不是很理解；
  3. 对.9图的使用方法仍旧存有疑惑；
