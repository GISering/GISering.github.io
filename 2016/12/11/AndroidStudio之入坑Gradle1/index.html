AndroidStudio之入坑Gradle(1)

发表于 2016-12-11   |   分类于 Android   |     |   阅读次数

来泰安学习已经有半个月了，这里的生活是准军事化的，不太习惯，总感觉形式大于内容，各种不合理的安排不再吐槽了，总之很有“特色”、总之“罄竹难书”。好在我懂得自个儿找乐子，于是就接着研究手头的Android项目，代码撸起来！！！Google已经停止对Eclipse Android开发工具的更新支持，同时AndroidStudio已经发布到V2.2.3，从颜值到各项体验远远超过Eclipse。对我来说，改变最大的就是构建工具的转变Ant->Gradle，因为我着实在Gradle上，掉进了很多坑，从下载速度感人到各种由于Gradle版本不一致造成的错误实在让我捶胸顿足。这两天一直在填坑，弄清楚了不少问题，所以我想是时候把坑踏平成套路了🤔。

这里先总结一下有关Gradle安装、配置以及在AnddroidStudio中我遇到的各类问题。

Gradle

简单认识

Gradle是基于Ant和Maven概念的项目自动化构建工具，使用一种基于Groovy的特定领域语言(DSL)来声明项目设置——Bing

Google选择Gradle作为AndroidStudio的构建工具而放弃Ant，从我浅薄的认识来说，Gradle综合了Ant和Maven的优点：不仅具备Ant的灵活性，也具备Maven的生命周期管理能力，而且它的构建脚本简单，有许多开箱即用的插件。

安装位置

作为中度强迫症患者，我喜欢弄清楚安装的每个软件都对我电脑做了什么！尤其是它们的安装位置和安装目录，不然心里不踏实😂。通过“找一找”发现，默认情况下Gradle会在两处安家：

  1. AndroidStudio的安装目录(A处)"安装路径/AndroidStudio/Gradle/"
  2. 用户的家目录(B处)在：
    • Mac: "/Users/username/.gradle/wrapper/dists"
    • Windows: "C:\Users\username\gradle\wrapper\dists“

开始填坑

为什么要在两处都安装Gradle?

下图是AS中Gradle配置窗口，默认选择官方也推荐选择“Use default gradle wrapper”：

  1. A处：默认情况下只有一个版本的Gradle，安装时间在是用AS建立第一个Project时下载的（这也是造成本地创建第一个项目时很慢的原因），版本号我判断应该是由我们所下载的AS版本决定的；当我们每次新建项目时，会根据该目录下的Gradle版本来创建Gradle配置文件，同样项目的Gradle wrapper版本也是由该版本决定的，其版本号具体配置在gradle-wrapper.properties文件中的distributionUrl字段中，可以看到这是一个URL地址。
  2. B处：如果你曾多次导入从别处获得的Android项目，你会发现B处的目录下会有多个版本的Gradle。当我们导入从开源社区下载的Android项目时，AS会选择根据默认项“Use default gradle wrapper”，即执行gradle wrapper的构建命令：首先判断B处是否存在在gradle-wrapper.properties文件中的distributionUrl字段中定义的Gradle版本，如果没有，就会根据URL地址下载到B处。这也是为什么我们每次导入项目时，会如此缓慢的原因。

    AS 是如何判断B处是否存在它需要的Gradle版本的?

但是我发现，AndroidStudio的安装目录下也有一个Gradle文件夹(这里称之为B处)，里面也是安装好的Gradle，这两处所安装的Gradle里的核心可执行文件是一致的没有区别，只是A出比B处多了doc和example等无关紧要的文件。既然如此，为什么还要放置两处呢？抱着不见棺材不掉泪的精神，我阅读了Intelli IDEA官方文档，终于明白了这样做的目的所在，更准确地说是找出了出现这种现象的缘由。
